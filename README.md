# かんじドリル（小4・書き取り）

小学4年生の漢字「書き取り」を、**間違えた字を中心に毎日10問**くりかえす iPad 向けの
シンプルな練習アプリです。サーバー・ログイン不要。HTML/CSS/JS だけで動きます。

## できること

- 例文の空欄に漢字を書く問題を毎日出題（既定10問）
- 紙でも、画面の手書きスペース（Apple Pencil可）でも書ける
- 「答えを見る」→ **自分で ○ / ✕ をタップ**（自己採点。字形は人が見る）
- 間違えた字は **翌日また最優先で出題**（間隔反復／Leitner方式）
- 連続日数・おぼえた字数・苦手リストを表示
- 学校のドリル/ノートの間違いを **Geminiで書き起こして一括取り込み**
- 進捗のバックアップ（書き出し／復元）

## 使い方（毎日）

1. ホームの「▶ はじめる」を押す
2. 例文を見て、空欄の漢字を書く（紙 or 画面）
3. 「答えを見る」→ 正解とくらべて ○ / ✕ をタップ
4. 10問おわると結果が出る。間違えた字は明日また出ます

## 間違えた問題の取り込み（写真 → Gemini → アプリ）

> ここが「うちの子が今つまずいている字」を最優先で出すための肝です。

1. **撮る**：学校の漢字ドリル/ノートで ✕ だった書き取りをスマホで撮影
2. **Geminiで書き起こす**：写真を添付し、下の「Gemini用プロンプト」を貼って送信
3. **貼り付ける**：返ってきたJSONを、アプリの「⚙ 問題のとりこみ」→「① はりつけてとりこむ」に貼って取り込む

取り込んだ字は `要復習` 扱いになり、**次の出題でいちばん先に**出ます。

### Gemini用プロンプト（コピペして使用）

```text
あなたは小学生の漢字学習をサポートするアシスタントです。
添付した写真は、小学4年生が使っている漢字ドリル／ノートの「書き取り（漢字を書く）問題」です。
この中から、子どもが間違えた問題だけを抽出してください。
「間違えた問題」とは、×印がついている・赤で直されている・空欄のまま・誤字になっている問題を指します。
判断に迷う場合は、その問題も含めてください（後で人が確認します）。

抽出した各問題を、次の形式の JSON 配列として出力してください。

[
  {
    "prompt": "目を{__}ます",
    "answer": "覚",
    "reading": "さ",
    "word": "覚ます"
  }
]

各フィールドの意味：
- prompt … 例文。漢字を書かせる箇所を必ず {__} に置き換える。
- answer … その箇所に入る正しい漢字（1〜複数字）。
- reading … 送りがな・読みのヒント（ひらがな）。
- word … 答え合わせで見せる完成した語（例：覚ます）。

ルール：
1. 写真から例文が読み取れない場合は、ドリルの語（熟語）をもとに prompt を作ってよい。
   その場合 prompt は word の漢字部分を {__} に置き換えた形にする（例：word「友達」→ prompt「{__}だち」）。
2. 出力は JSON 配列のみ。前後に説明文・見出し・```（コードブロック記号）を付けない。
3. 必ず有効な JSON にする（全角の引用符やカンマ抜けを使わない）。
4. 同じ漢字が複数あっても、写真にある分はすべて1問ずつ出力する。
5. 読みや漢字が確実に判別できないものは出力に含めない。
```

> アプリ側は ```（コードブロック記号）や前後の説明文が混ざっていても、JSON配列だけ取り出して
> 取り込めるようにしてあります。多少の余分があってもそのまま貼り付けてOKです。

## iPad で「アプリのように」使う

1. このフォルダを Web に公開（GitHub Pages など）するか、Mac で簡易サーバーを立てる
   ```bash
   cd kanji_drill
   python3 -m http.server 8000
   # 同じWi-FiのiPadから http://<MacのIP>:8000 を開く
   ```
2. iPad の Safari で開く → 共有ボタン → **「ホーム画面に追加」**
3. 以後はホームのアイコンから起動（オフラインでも動作）

> 注意：`file://`（ファイルを直接開く）ではオフライン機能（service worker）が動きません。
> かならず `http://` または `https://` で開いてください。

## データについて

- 学習のきろくは **その端末の Safari（localStorage）だけ** に保存されます
- 機種変更や Safari データ削除の前に、「③ バックアップ」から **書き出し** をおすすめします
- 別端末へは「書き出したJSON」を「よみこむ（復元）」で移せます

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | 画面の骨組み |
| `style.css` | 見た目（子ども向け・大きめ） |
| `app.js` | 出題ロジック・間隔反復・自己採点・取り込み・保存 |
| `kanji_data.js` | スターター問題（4年配当の書き取り。穴埋め用） |
| `manifest.webmanifest` / `sw.js` | PWA（ホーム追加・オフライン） |
| `icons/` | ホーム画面アイコン |
